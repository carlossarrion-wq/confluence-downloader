  2. Se recuperan todas las lecturas (entidad IMAGE) de los últimos 7 días de la distribuidora indicada en datalake_distribuidora (valor actual Nedgia) y se lanza la consulta que recupera toda la información indicada en el punto anterior en el campo datalake_campos

  3. Se genera un fichero csv con el resultado de la consulta y teniendo en cuenta:

    1. nombre del informe (datalake_nombre_fichero)

    2. cabecera del informe (datalake_cabecera)

    3. Archivo CSV con codificación Latin1, separador punto y coma (;), valores entrecomillados y sin compresión

  4. Se deja el fichero generado en el S3 creado para que un proceso posterior lo recoja y lo cargue en el Data lake. Se está considerando:

    1. bucket y ruta donde ha que dejarlo (datalake_nombre_bucket y datalake_ruta_bucket)

    2. el informe se deja con el nombre: valor de datalake_nombre_fichero concatenado con YYYYMMDD.csv

    3. credenciales para conectarse a AWS (datalake_clave_acceso_secreta )
