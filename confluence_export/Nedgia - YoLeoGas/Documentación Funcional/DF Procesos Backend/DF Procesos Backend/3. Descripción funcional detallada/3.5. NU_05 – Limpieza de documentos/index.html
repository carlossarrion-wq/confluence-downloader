Espacio: Y

El proceso CleanStorage se encarga de eliminar del storage de Azure aquellas imágenes con una antigüedad superior a x días. El valor a considerar viene determinado por el parámetro cfg_documentum_retain_image_days de la entidad CONFIGURATION y actualmente tiene el valor de 180 días. Este valor se puede modificar desde la web de backoffice, ver NU_09 - Configuración global.

El proceso realiza lo siguiente:

  1. Buscar imágenes que hay que borrar. Son aquellas imágenes de IMAGE que cumplen:

    • Ya han sido enviadas correctamente a distribuidora (image_status_stage = 'PROCESSED')

    • No han sido borradas todavía (image_released = 0)

    • Son más antiguas que lo indicado por el parámetro cfg_documentum_retain_image_days de la entidad CONFIGURATION

    • Están en el contenedor “doonamis”

    • Se procesan un máximo de 500 imágenes por ejecución

  2. Eliminar imágenes. Para cada imagen encontrada en el paso anterior:

    • Conecta al Azure Blob Storage

    • Busca el archivo físico (ej: abc-123.jpg)

    • ELIMINA el archivo del almacenamiento

    • Libera el espacio ocupado

  3. Marca la imagen como "liberada". En la entidad IMAGE para cada imagen:

    1. Cambia image_released de false (0) a true (1)

    2. No elimina el registro de la base de datos
