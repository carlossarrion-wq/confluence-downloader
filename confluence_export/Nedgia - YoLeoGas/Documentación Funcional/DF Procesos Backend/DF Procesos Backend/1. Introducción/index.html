1.1.      Objetivos

El objetivo de este documento es detallar el funcionamiento de los procesos de backend que gestionan las lecturas realizadas desde la aplicación móvil Yo Leo Gas y la aplicación web de backoffice.

1.2.      Glosario

  • Nedgia: distribuidora de gas del grupo Naturgy. Solo se considerarán contadores de gas de esta distribuidora

  • CUPS: Código Único del Punto de Suministro, identificador único del punto de suministro

  • Zeus: backend donde están registrados todos los CUPS de gas de Nedgia

  • App YoLeoGas: aplicación móvil de Nedgia para que los titulares de contratos de gas de esta distribuidora puedan enviar la lectura del contador realizando una fotografía del contador o introduciéndola manualmente si no consiguen realizar una fotografía de calidad. Esta app también la utilizan los técnicos de operaciones para dar la lectura.

  • Web de backoffice: aplicación web desde donde el equipo de backoffice de Nedgia puede validar o descartar las lecturas que el OCR no ha podido procesar correctamente o no ha superado el índice de confianza

  • OCR: componente que de una fotografía de un contador de gas, extrae el valor numérico de la lectura que se muestra en la fotografía.

  • Cliente: usuario de la aplicación YoLeoGas que haciendo una foto envía la lecturas a Nedgia
