1. Resumen / Propósito

Este plan de pruebas define los casos de prueba necesarios para validar la web de Backoffice de Yo Leo Gas.

2. Objetivos de las pruebas

  • Verificar el correcto funcionamiento de los flujos críticos.

3. Alcance

Excluye:

  • El alta de los usuarios que pueden entrar se tiene que solicitar vía F1

4. Entorno de pruebas

  • Desarrollo: https://backoffice-yoleogas-des.nedgia.es/

  • Integración/ Preproducción: https://backoffice-yoleogas-pre.nedgia.es/

3

Validación lecturas - Proceso

Usuario entra en la pantalla y valida la lectura enviada al móvil

La lectura pasa a ser enviada a la distribuidora y de la entidad IMAGE se actualiza lo siguiente:

  • image_status_stage → pasa a tener el valor PENDING_SGC_PROCESSING

  • image_date_processed_manual: fecha en la que ha validado manualmente la lectura

  • image_meter_reading_value_manual: valor de la lectura que se ha dado cuando se ha validado manualmente

  • user_id: usuario que ha validado manualmente la lectura

Ya no aparece la lectura como pendiente de validar.

4

Validación perfiles - Proceso

Usuario entra en la pantalla, introduce un valor en lectura introducida por el supervisor, en el número de serie y pulsa el botón “Corregir/Actualizar/Modificar”

  • De la entidad PROFILE se actualizan:

    • pfr_status_stage → pasa a tener el valor PROCESSED

    • prf_meter_serial_value_manual: número de serie del contador que se ha introducido manualmente

  • De la entidad IMAGE se actualiza:

    • image_status_stage → pasa a tener el valor PENDING_SERVER_PROCESSING

  • Ya no aparece el perfil como pendiente de validar.

    • pfr_status_stage → pasa a tener el valor DISCARDED

  • De la entidad IMAGE se actualiza

    • image_status_stage → pasa a tener el valor DISCARDED.
