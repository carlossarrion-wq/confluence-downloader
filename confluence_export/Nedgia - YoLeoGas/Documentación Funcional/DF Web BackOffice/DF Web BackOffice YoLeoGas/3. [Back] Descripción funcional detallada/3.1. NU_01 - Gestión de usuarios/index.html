Cada usuario de la web de Backoffice tiene asociado un rol específico que define sus permisos y las pantallas a las que puede acceder.

3.1.1. Tipología de usuarios

Actualmente existen 3 roles en la aplicación:

Administrador (user_type = 0):

  • Tiene acceso a todas las funcionalidades de la aplicación

Manager (user_type = 3):

  • Tiene acceso a todas las funcionalidades, excepto la configuración global

  • Puede validar imágenes y perfiles que hayan pasado a supervisión

Validador (user_type = 1):

  • Su función principal es la validación de imágenes y perfiles

  • Puede ver la lista de imágenes y perfiles

  • Puede reasignar perfiles

  • No tiene acceso a la configuración, informes ni consulta de usuarios

3.1.3. Flujo de inicio de sesión

Al abrir la aplicación se solicitará la introducción de una cuenta de usuario y clave de paso. Esta securización se realizará por Entra ID de Microsoft con la plataforma corporativa de Naturgy.

3.1.4. Gestión de usuarios y roles

La gestión de usuarios y roles está delegada a Entra ID. Para la creación de un nuevo usuario o modificación de su rol se deberá crear una solicitud de F1.

En EntraID para cada usuario estarán informados los siguientes atributos:

  • nombre y apellidos

  • código de usuario

  • rol que el usuario tiene en la aplicación

Esta gestión quedará eliminada de la aplicación, que verificará el rol existente tanto en la parte de frontal de usuario para mostrar las funciones como la parte de API para validar los accesos correspondientes.

3.1.5. Login de usuarios

Cada vez que un usuario entre en la aplicación web de backoffice la entidad USER se informará de la siguiente forma:

  • user_id: código del usuario que proporciona EntraID

  • user_name: nombre y apellidos del usuario que proporciona EntraID

  • user_lastlogin: fecha en la que se está accediendo

  • user_type: rol que el usuario tiene en la aplicación que proporciona EntraID

Toda esta información se actualizará en la tabla portaluser.
