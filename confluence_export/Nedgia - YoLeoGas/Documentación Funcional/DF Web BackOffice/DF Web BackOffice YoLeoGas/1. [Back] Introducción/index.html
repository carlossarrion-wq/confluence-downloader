1.1 Objetivos

El objetivo de este documento es detallar el funcionamiento de la aplicación de backoffice de Yo Leo Gas.

En la aplicación de backoffice de Yo Leo Gas se pueden consultar todas las imágenes y lecturas que se han realizado desde la app Yo Leo Gas. También se pueden gestionar aquellas lecturas que el OCR no ha sabido identificar correctamente, así como consultar informes.

El detalle de todas las pantallas se pueden encontrar en figma:

1.2 Glosario

  • Nedgia: distribuidora de gas del grupo Naturgy, solo se considerarán contadores de gas de esta distribuidora

  • CUPS: Código Único del Punto de Suministro, identificador único del punto de suministro

  • Zeus: backend donde están registrados todos los CUPS de gas de Nedgia

  • OCR: componente que con una fotografía de un contador de gas, extrae el valor numérico de la lectura que se muestra en la fotografía

  • Cliente: usuario de la aplicación YoLeoGas que envía lecturas a Nedgia, se procesan por el sistema y aparecen en el sistema backoffice de YoLeoGas.

  • Usuario: colaborador de Naturgy o empresa externa que se conecta al sistema backoffice de YoLeoGas para gestión o consulta de las lecturas enviadas por el cliente.

  • Aplicación móvil YoLeoGas: aplicación móvil de Nedgia para que los clientes de esta distribuidora puedan enviar la lectura del contador realizando una fotografía del contador o manualmente si no consiguen realizar una fotografía de calidad

  • Aplicación móvil W&T: aplicación móvil que los técnicos de gas utilizan cuando van a visitar a los clientes a su domicilio para una inspección periódica de gas. Aprovechando esta visita, los técnicos realizan una fotografía del contador desde esta aplicación que se integra con la app YoLeoGas.
