YoleoGas OCR App API - Diseño Técnico

📋 Información General

Tipo de Proyecto

Arquitectura de Microservicios desarrollada en C# .NET 8.0 para el sistema completo de gestión de lecturas de contadores de gas YoleoGas con funcionalidades de OCR (Optical Character Recognition). Esta solución integra una API REST principal junto con múltiples Azure Functions especializadas para el procesamiento automatizado de imágenes, notificaciones, historización y integración con sistemas externos.

Propósito

Sistema integral para la captura, procesamiento OCR, validación y gestión de lecturas de contadores de gas mediante dispositivos móviles. Incluye procesamiento automatizado de imágenes, envío de notificaciones push, integración con distribuidores de gas y gestión completa del ciclo de vida de las lecturas.

🏗️ Arquitectura del Sistema

Patrón Arquitectónico

  • Microservicios Serverless con Azure Functions

  • API Gateway Pattern con API REST central

  • Event-Driven Architecture para procesamiento de imágenes

  • Repository Pattern para acceso a datos

  • Dependency Injection nativo de .NET

  • Service Layer para lógica de negocio

Componentes Principales

1. YoleoGas.API (API REST Principal)

  • Puerto de Entrada: Punto de acceso principal para aplicaciones móviles

  • Controladores REST: Gestión de perfiles, imágenes, dispositivos, CUPS, NIF

  • Health Checks: Monitoreo de estado de la aplicación

  • Swagger/OpenAPI: Documentación automática de la API

2. YoleoGas.Shared (Librería Compartida)

  • Modelos de Datos: Entidades de base de datos (EF Core)

  • DbContext: Configuración de Entity Framework

  • Utilidades Comunes: Servicios compartidos entre componentes

  • Validaciones: Lógica de validación centralizada

3. Azure Functions (Microservicios Especializados)

YoleoGas.BatchOcr

  • Propósito: Procesamiento OCR de imágenes en lotes

  • Trigger: Timer/Queue

  • Funciones: Análisis de imágenes, extracción de lecturas, validación

YoleoGas.PushServer

  • Propósito: Envío de notificaciones push

  • Integración: Firebase (Android), Apple Push (iOS)

  • Funciones: Gestión de tokens, envío masivo de notificaciones

YoleoGas.NotificationSender

  • Propósito: Gestión y envío de notificaciones generales

  • Trigger: Queue/HTTP

  • Funciones: Procesamiento de plantillas, envío personalizado

YoleoGas.NotificationsDownloader

  • Propósito: Descarga de notificaciones de sistemas externos

  • Procesadores: Distribuidor genérico, GNF específico

  • Funciones: Sincronización automática de notificaciones

YoleoGas.Historize

  • Propósito: Historización de datos

  • Trigger: Timer/Queue

  • Funciones: Archivado de registros, limpieza de datos

YoleoGas.NedgiaReadingSender

  • Propósito: Envío de lecturas a sistemas Nedgia

  • Integración: APIs externas de distribuidores

  • Funciones: Formateo de datos, envío seguro

YoleoGas.BatchDoc

  • Propósito: Procesamiento de documentos en lote

  • Funciones: Gestión de documentos relacionados con lecturas

YoleoGas.BatchError

  • Propósito: Gestión y procesamiento de errores en lote

  • Funciones: Recuperación automática, logging avanzado

🚀 API Endpoints Principales

Gestión de Imágenes

  • POST /api/images/upload - Subida de imágenes de contadores

  • GET /api/images/{id} - Obtener imagen específica

  • POST /api/images/{id}/process - Procesar imagen con OCR

  • PUT /api/images/{id}/reading - Actualizar lectura manual

Gestión de Perfiles

  • GET /api/profiles - Lista de perfiles con filtros

  • POST /api/profiles - Crear nuevo perfil de contador

  • GET /api/profiles/{id} - Detalle de perfil específico

  • PUT /api/profiles/{id} - Actualizar perfil

  • DELETE /api/profiles/{id} - Eliminar perfil

Gestión de Dispositivos

  • POST /api/devices/register - Registrar nuevo dispositivo

  • PUT /api/devices/{id} - Actualizar información de dispositivo

  • POST /api/devices/{id}/token - Actualizar token push

Consulta de CUPS

  • GET /api/cups/{cups} - Buscar información por CUPS

  • GET /api/cupsv2/search - Búsqueda avanzada de CUPS

  • POST /api/cups/validate - Validar CUPS

Consulta de NIF

  • GET /api/nif/{nif} - Buscar información por NIF

  • POST /api/nif/validate - Validar NIF

Configuración y Parámetros

  • GET /api/params - Obtener parámetros de configuración

  • GET /api/params/distributor/{id} - Parámetros por distribuidor

Integración Hermes

  • POST /api/hermes/images - Procesar imágenes con Hermes OCR

  • GET /api/hermes/status/{id} - Estado de procesamiento Hermes

Health Checks

  • GET /health - Health check básico con estado de BD

  • GET /health/ready - Health check de disponibilidad

  • GET / - Endpoint raíz con información de servicio
