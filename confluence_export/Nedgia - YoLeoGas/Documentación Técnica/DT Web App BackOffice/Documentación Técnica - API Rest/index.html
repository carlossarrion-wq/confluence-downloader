📋 Información General

Tipo de Proyecto

API REST Backend desarrollada en C# .NET 8.0 para el sistema de gestión de lecturas de contadores de gas YoleoGas. Esta API forma parte de la arquitectura de microservicios del ecosistema YoleoGas, proporcionando servicios de backoffice para la gestión y validación de lecturas y perfiles de contadores.

Propósito

Sistema de gestión empresarial para validación manual de lecturas OCR, administración de perfiles de contadores, generación de informes estadísticos y gestión de imágenes asociadas a las lecturas.

🔐 Seguridad

Gestión de Secretos

  • Todos los accesos y passwords a elementos de terceros están guardados en almacenecs de claves Azure.

  • Las APIs tienen configuración CORS Activa para solo permitir llamadas desde los frontales relacionados con cada entorno.

  • El API está securizado por EntraID dónde se definen los usuarios y roles asignados a la aplicación.

🚀 API Endpoints Principales

Gestión de Lecturas

  • GET /api/v1/lecturas - Lista paginada con filtros

  • GET /api/v1/lecturas/{id} - Detalle de lectura

  • PUT /api/v1/lecturas/{id} - Actualizar lectura

  • POST /api/v1/lecturas/{id}/validar - Validar lectura

Gestión de Perfiles

  • GET /api/v1/perfiles - Lista paginada con filtros

  • GET /api/v1/perfiles/{id} - Detalle de perfil

  • PUT /api/v1/perfiles/{id} - Actualizar perfil

  • POST /api/v1/perfiles/{id}/validar - Validar perfil

Gestión de Imágenes

  • GET /api/v1/imagenes/{id} - Obtener imagen

Informes y Estadísticas

  • GET /api/v1/informes/cluster - Informes de cluster

  • GET /api/v1/informes/gestion - Informes de gestión

  • GET /api/v1/lecturas/estadisticas - Estadísticas de lecturas

  • GET /api/v1/perfiles/estadisticas - Estadísticas de perfiles

Configuración

  • GET /api/v1/configuration - Configuración global

  • GET /api/v1/distributor-configuration - Configuración por distribuidor

Health Checks

  • GET /health - Health check básico

  • GET /api/v1/health - Health check detallado

  • POST /api/v1/health/database-connection - Validación completa de BD

🏥 Health Checks y Monitoreo

Health Checks Disponibles

Se habilitan tres servicios de Health Check funcional que servirán para el alertado de funcionamiento de la infraestructura.

  1. Basic Health Check (/health)

    • Endpoint básico de infraestructura

    • Verificación rápida de estado de la API

  2. Detailed Health Check (/api/v1/health)

    • Información detallada del sistema

    • Verificación de servicios críticos

  3. Database Connection Health Check (/api/v1/health/database-connection)

    • Validación completa de conectividad a BD

    • Fallback automático Key Vault → App Settings

    • Métricas de tiempo de respuesta por paso
