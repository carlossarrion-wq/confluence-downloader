1. INFORMACIÓN GENERAL DEL PROYECTO

1.1 Tipo de Proyecto

  • Aplicación Web Frontend: Single Page Application (SPA)

  • Propósito: Plataforma de gestión backoffice para YoleoGas

  • Arquitectura: Client-side application con comunicación API REST

1.2 Identificación

  • Nombre: YoleoGas Backoffice Frontend

  • Versión: 1.0.0

  • Repositorio: AZU-NED-OCR-YOLEOGAS-SERVER/yoleogas-backoffice/yoleogas-backoffice-front

  • Tipo: Aplicación React TypeScript con Vite

2. STACK TECNOLÓGICO

2.1 Tecnologías Principales

  • Framework JavaScript: React 18.2.0

  • Lenguaje: TypeScript 5.8.3

  • Build Tool: Vite 7.1.7

  • Bundler: Rollup (integrado en Vite)

  • Package Manager: npm

2.2 Dependencias Core

  • @vitejs/plugin-react: 4.3.1 - Plugin oficial de React para Vite

  • react-dom: 18.2.0 - Renderizado DOM para React

2.3 Frameworks UI y Styling

  • CSS Framework: Tailwind CSS 3.4.0

  • UI Components: Radix UI (múltiples componentes)

    • Dialog, Dropdown Menu, Select, Popover, Tooltip

    • Navigation Menu, Scroll Area, Progress

    • Checkbox, Label, Separator, Tabs, Slot

  • Iconos: Lucide React 0.542.0

  • Utilidades CSS:

    • clsx 2.1.1

    • tailwind-merge 3.3.1

    • class-variance-authority 0.7.1

2.4 Bibliotecas de Estado y Datos

  • Data Table: @tanstack/react-table 8.21.3

  • Date Handling: date-fns 4.1.0

  • Calendar: react-day-picker 9.9.0

2.5 Herramientas de Desarrollo

  • Linting: ESLint 9.33.0 con TypeScript ESLint 8.39.1

  • PostCSS: autoprefixer 10.4.21

  • Build Types: @types/node, @types/react, @types/react-dom

2.6 Notificaciones y UX

  • Toast Notifications: Sonner 2.0.7

  • Theme Management: next-themes 0.4.6

3. AUTENTICACIÓN Y SEGURIDAD

3.1 Sistema de Autenticación

  • Proveedor: Microsoft Azure Active Directory (Azure AD)

  • Biblioteca: Microsoft Authentication Library (MSAL)

    • @azure/msal-browser 4.25.0

    • @azure/msal-react 3.0.20

3.2 Configuración de Seguridad

  • Auth Flow: Authorization Code Flow con PKCE

  • Scopes: User.Read (Microsoft Graph)

  • Cache: sessionStorage para mayor seguridad

  • Redirect Handling: navegación automática post-login

6. INTEGRACIÓN CON SERVICIOS EXTERNOS

6.1 API Backend

  • Base URL: Configurable por entorno via VITE_BACKOFFICE_API_URL

  • Protocolo: HTTP/HTTPS REST API

  • Formato: JSON

  • Autenticación: Bearer Token (Azure AD)

6.2 Servicios API

  • Lecturas API: Gestión de lecturas de contadores

  • Perfiles API: Gestión de perfiles de usuario

  • Informes API: Generación y consulta de informes

  • Estadísticas API: Métricas y analytics

  • Configuration API: Configuración de distribuidoras

6.3 Microsoft Graph API

  • Endpoint: https://graph.microsoft.com/v1.0/me

  • Uso: Obtención de información del usuario autenticado

  • Scopes: User.Read
