Arquitectura General

YoLeoGas es una solución distribuida en Microsoft Azure que gestiona la lectura de contadores de gas y servicios relacionados. La arquitectura se compone de:

  • 1 API REST principal (Azure App Service)

  • 1 Aplicación BackOffice (Azure Cloud Service - WebRole)

  • 4 Azure Functions para procesamiento asíncrono

  • 11 WebJobs organizados en 2 grupos de notificadores

  • 3 Librerías compartidas para lógica común

5. NTG.YoLeoGas.PushServer ✅

Función: Servicio de notificaciones push que procesa mensajes de Service Bus y envía notificaciones a dispositivos móviles iOS y Android.

Funcionalidad

  • Gestión de dispositivos: Identificación automática de plataforma y envío dirigido

  • Actualización de estado: Marca las notificaciones como procesadas en base de datos

  • Logging integrado: Registro detallado de operaciones y errores

Tecnologías Utilizadas

  • Microsoft.Azure.Functions.Worker 1.4.0 (Isolated Worker Model)

  • HttpClient para llamadas HTTP a servicios externos

Arquitectura del Servicio

Trigger Principal:

  • PushServer.Run(): Function triggered por Service Bus Queue

  • Queue: %NotificationQueueName% (configurado via Key Vault)

  • Connection: ServiceBusConnectionString (configurado via Key Vault)

Clases Especializadas:

  • FirebaseSender: Envío de notificaciones a dispositivos Android via Firebase

  • AppleSender: Envío de notificaciones a dispositivos iOS via Pushwoosh

  • FirebaseMessage: Modelo para mensajes Firebase

  • Message: Modelo completo para notificaciones iOS con 10 parámetros personalizables

Flujo de Procesamiento

  1. Recepción: Message desde Service Bus con formato mensaje$guid

  2. Parsing: Extracción del NotificationId del mensaje

  3. Consulta DB: Obtención de datos de notificación y perfil asociado

  4. Identificación dispositivo: Consulta del dispositivo asociado al perfil

  5. Envío dirigido:

    • Android: Firebase Cloud Messaging

    • iOS: Pushwoosh API

  6. Actualización: Marca NotificationProcessingDate en base de datos

Integración con Servicios Externos

Firebase Cloud Messaging (Android):

  • Endpoint: Variable Firebase-url desde Key Vault

  • Autenticación: API Key via FirebaseSender-API-KEY

  • Formato: JSON con to (device token) y notification (body)

Pushwoosh (iOS):

  • Endpoint: Variable Pushwoorsh-url desde Key Vault

  • Autenticación: Pushwoosh-Auth y Pushwoosh-App

  • Características: Soporte para badges, scheduling, timezone ignore

  • Protocolo: HTTPS con TLS 1.2

Modelo de Datos

Entidades Principales:

  • Notification: Notificación con mensaje, fecha creación, fecha procesamiento

  • Profile: Perfil de contador asociado (PrfId, PrfCups)

  • Device: Dispositivo móvil (Platform, DeviceToken, PrfId)

Plataformas Soportadas:

  • android-fcm: Android con Firebase Cloud Messaging

  • ios: iOS con Apple Push Notification via Pushwoosh
