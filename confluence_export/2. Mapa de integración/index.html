El diagrama muestra el flujo de datos completo del sistema de validación de lecturas de gas:

  1. App Móvil → captura lecturas de contadores y las envía

  2. Motor OCR → procesa las imágenes y extrae datos automáticamente

  3. Base de Datos → almacena las lecturas procesadas para validación

  4. API Backoffice → consulta datos y sirve al frontend para validación manual

  5. Frontend → interfaz donde los operadores validan las lecturas

Flujos paralelos:

  • Las imágenes se almacenan en Blob Storage desde el Motor OCR

  • La API descarga imágenes directamente desde Blob Storage para mostrarlas en el frontend

Los procesos de BackOffice que gestionan la información no son alcance de este documento
